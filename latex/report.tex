\begin{abstract}
Link prediction in academic collaboration networks is a fundamental task in graph machine learning. This project applies and compares three methods—Common Neighbors (heuristic), MLP (feature-based), and Graph Convolutional Networks (GCN, message-passing)—on the OGB-collab dataset. Experiments are conducted at three data scales (10\%, 50\%, 100\%) with multi-seed evaluation to ensure statistical robustness. The GCN achieves the best predictive performance (Hits@50 = 0.457 on average at full scale, best seed = 0.484), significantly outperforming MLP (0.181) but falling short of the Common Neighbors heuristic (0.515).
\end{abstract}

%-----------------------------------------------------------------
\section{Motivation and Goals}
%-----------------------------------------------------------------

Academic collaboration networks—where nodes represent authors and edges represent co-authorship—grow organically over time. Predicting which authors will collaborate in the future has practical applications in research recommendation systems, funding allocation analysis, and understanding the dynamics of scientific communities.

The OGB-collab dataset~\cite{ogb} provides a standardized benchmark for this task with a temporal split: training edges from before 2010, validation edges from 2010, and test edges from 2011 onwards. This mirrors real-world deployment where the model must generalize to future collaborations.

\textbf{Goals:}
\begin{itemize}[nosep]
    \item Predict missing edges on the OGB-collab dataset using three complementary methods.
    \item Compare a simple heuristic (Common Neighbors), a feature-only neural network (MLP), and a graph neural network (GCN).
    \item Measure predictive performance (Hits@50, Hits@100) across three data scales (10\%, 50\%, 100\%).
    \item Conduct multi-seed evaluation (3 seeds per configuration) to quantify variance and ensure statistical reliability.
\end{itemize}

\textbf{Implementation Environment:}
\begin{itemize}[nosep]
    \item Language: Python 3.11
    \item Libraries: PyTorch, PyTorch Geometric, OGB, NumPy, Plotly, Streamlit
    \item GPU: NVIDIA RTX 5060 Ti (16 GB VRAM)
    \item OS: Linux (Fedora 43)
\end{itemize}

\subsection{Data Scaling}

To study how performance scales with data volume, three fractions of the training edges are used while keeping validation and test sets fixed:
\begin{itemize}[nosep]
    \item \textbf{Scale 0.1}: 10\% of training edges (118K edges)
    \item \textbf{Scale 0.5}: 50\% of training edges (589K edges)
    \item \textbf{Scale 1.0}: 100\% of training edges (1.18M edges)
\end{itemize}

This design tests whether message-passing methods (like GCN) need many training edges to be effective, or whether feature-based methods can perform well with limited data.

\section{Methods}
%-----------------------------------------------------------------

\subsection{Common Neighbors (Baseline Heuristic)}

The Common Neighbors heuristic~\cite{newman2001} is a parameter-free baseline that scores a candidate edge $(u,v)$ as:

\begin{equation}
    \text{score}(u,v) = |\mathcal{N}(u) \cap \mathcal{N}(v)| + \varepsilon
\end{equation}

where $\mathcal{N}(u)$ is the neighbor set of node $u$ in the training graph and $\varepsilon = 10^{-6}$ is a small tie-breaker. The graph is symmetrized before neighbor counting. Despite its simplicity, Common Neighbors is a remarkably strong baseline—often competitive with learned methods on collaboration graphs.

\textbf{Complexity:} $O(1)$ per candidate edge after adjacency list construction. No training required.

\subsection{MLP (Feature-Only Baseline)}

The MLP method ignores graph structure entirely and predicts links based solely on node features. Given feature vectors $\mathbf{x}_u$ and $\mathbf{x}_v$, the Hadamard product is computed:

\begin{equation}
    \mathbf{h} = \mathbf{x}_u \odot \mathbf{x}_v
\end{equation}

This 128-dimensional vector is passed through a 3-layer MLP with hidden dimension 256 and sigmoid output. Training uses binary cross-entropy loss with negative sampling (1:1 ratio).

\textbf{Hyperparameters:} hidden\_channels=256, num\_layers=3, dropout=0.0, lr=0.01, batch\_size=65,536, epochs=200.

\subsection{GCN (Graph Convolutional Network)}

The GCN model~\cite{kipf2017} consists of two components:

\textbf{GCN Encoder:} A 3-layer graph convolutional network maps node features to node embeddings via message passing over the training graph.

\textbf{Link Predictor:} Pairs of node embeddings are combined via Hadamard product and passed through a 3-layer MLP with sigmoid output, identical in structure to the standalone MLP predictor.

Training uses full-batch message passing with binary cross-entropy loss and negative sampling. All 200 epochs are run fully—no early stopping based on validation metrics. A Cosine Annealing scheduler (eta\_min=$10^{-6}$) smoothly decays the learning rate.

\textbf{Hyperparameters:} hidden\_channels=256, num\_layers=3, dropout=0.2, lr=0.005, batch\_size=65,536, epochs=200.

\subsection{Multi-Seed Evaluation}

To quantify variance and avoid overfitting to a lucky random seed, each method$\times$scale configuration is evaluated with 3 different random seeds (42, 123, 456). The final metrics are reported as mean $\pm$ standard deviation.

\section{System Architecture}
%-----------------------------------------------------------------

The project is organized into modular components for reproducibility and extensibility:

\begin{itemize}[nosep]
    \item \textbf{Data Module:} Automatic OGB-collab download; configurable data scaling; stratified edge splits respecting temporal ordering.
    \item \textbf{Methods Module:} Independent implementations for Common Neighbors, MLP, and GCN—each exposing a uniform \texttt{train()} / \texttt{evaluate()} interface.
    \item \textbf{Experiment Runner:} Orchestrates full pipelines; handles configuration, device selection (GPU/CPU), multi-seed execution, result serialization to JSON.
    \item \textbf{Evaluation Module:} Computes Hits@K metrics using OGB's evaluator; tracks runtime and memory; logs per-epoch validation metrics.
    \item \textbf{Multi-Seed Benchmark Script:} Runs all (method, scale, seed) combinations, aggregates results, exports CSV summaries and Plotly visualizations.
    \item \textbf{Streamlit Dashboard:} 6-tab interactive UI (Proje Özeti, Veri Seti Keşfi, Metodoloji, Deney Sonuçları, Sonuç \& Tartışma, İnteraktif Demo).
\end{itemize}

\section{Discussion}
%-----------------------------------------------------------------

\subsection{Best Method by Data Scale}

\textbf{At scale 0.1:} GCN is the clear winner (Hits@50 = 0.228 mean, 0.253 best), providing roughly 3$\times$ improvement over Common Neighbors (0.075). At this scale, the training graph covers only 10\% of collaborations—too sparse for structural heuristics. Node features (128-dim paper embeddings) carry enough signal for GCN to make useful predictions.

\textbf{At scale 0.5:} GCN and Common Neighbors are effectively tied (0.406 vs.\ 0.395), with a slight edge to GCN. This is the crossover point where structural information becomes roughly as informative as node features.

\textbf{At scale 1.0:} Common Neighbors takes the lead (0.515 vs.\ 0.457). With 1.18M training edges, the collaboration graph's structural patterns (shared co-authors) are so rich that a simple heuristic outperforms learned models.

\subsection{Why Common Neighbors Wins at Full Scale}

This is a well-documented phenomenon in link prediction~\cite{zhang2020}: academic collaboration graphs exhibit strong homophily and power-law degree distributions, making local heuristics remarkably effective. Common Neighbors captures the ``shared co-author'' pattern—researchers who share many collaborators are likely to collaborate themselves. With abundant training data, this signal dominates.

\subsection{Method Trade-offs}

\begin{itemize}[nosep]
    \item \textbf{Common Neighbors:} Zero training cost, instant inference ($\sim$1.4s), no need for node features. Fails completely at low data scales.
    \item \textbf{MLP:} Fast to train (seconds per epoch), uses features, but fundamentally limited by ignoring graph structure. Hits@50 plateaus at 0.181.
    \item \textbf{GCN:} Best at low/medium data scales, combines structure + features. Training is expensive ($\sim$19 min at full scale on GPU).
\end{itemize}

\section{Technical Challenges}
\label{sec:challenges}
%-----------------------------------------------------------------

\subsection{Other Challenges}

\begin{itemize}[nosep]
    \item \textbf{Multi-seed benchmark interference:} Two concurrent benchmark processes halve effective GPU speed (105s/run $\rightarrow$ 210s/run) due to CUDA context switching. A pre-flight check kills ghost processes.
    \item \textbf{PYTHONPATH issues:} Running scripts requires the project root in \texttt{sys.path}. This fix is silently lost on \texttt{git checkout}. Using \texttt{python -m} avoids this entirely.
    \item \textbf{Memory measurement noise:} Python's \texttt{psutil}-based tracking can produce negative deltas when the garbage collector runs between snapshots. This is expected behavior.
\end{itemize}
